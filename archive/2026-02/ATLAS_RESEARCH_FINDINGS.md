# ATLAS Marketing Skill — Comprehensive API Research Findings

**Research Date:** February 18, 2026  
**Scope:** Multi-platform marketing automation (TikTok, Instagram, YouTube, LinkedIn, Twitter, Reddit, Facebook)  
**Status:** Ready for implementation

---

## EXECUTIVE SUMMARY

Atlas will be built as a multi-platform content marketing agent with:
- **7 social platforms** via direct APIs or proxy services
- **3 image generation providers** (OpenAI, Stability, Replicate for flexibility)
- **Video generation** (Runway ML for AI-to-video)
- **Audio generation** (ElevenLabs for voice-overs)
- **Music licensing** (Epidemic Sound or platform native audio)
- **Posting infrastructure** (Postiz for cross-platform scheduling + analytics)
- **Hashtag research** (TikTok Search API, Instagram Graph API, TickerTrends)
- **Analytics aggregation** (platform-native APIs + optional third-party unification)

**Total Monthly API Cost (estimated):** $200-400/mo (depends on volume)  
**Build Timeline:** 8-10 weeks  
**Complexity:** Medium-High (multiple OAuth flows, async job handling)

## PART 1: PLATFORM-BY-PLATFORM ANALYSIS

### 1. TikTok

**Official APIs Available:**
- TikTok Business API (creator marketplace, content publishing)
- TikTok Ads API (campaign management)
- TikTok Creator Marketplace API (influencer collaboration)
- TikTok Search API + Discovery API (content discovery, hashtag analytics)

**Content Publishing Capability:**
- ✅ **Direct video upload via API** (as of 2024, through Adobe and partner integrations)
- ✅ **Scheduled posting** via integrations
- ✅ **Draft creation** (posts go to SELF_ONLY drafts, require manual approval + audio selection)

**Analytics:**
- ✅ TikTok Analytics API provides: views, likes, comments, shares, engagement rate
- ⚠️ Limited breakdown by content type (not as granular as Instagram)
- Metrics available: Creator Fund eligibility, monthly earnings (if enrolled)

**Hashtag Research:**
- TikTok Search API (official) — trend tracking
- TickerTrends (third-party) — historical hashtag data + trending analysis
- Native TikTok Discover page (manual, but data available through scraping if compliant)

**Rate Limits & Costs:**
- API access: **Free** (requires approved developer account + business account)
- TikTok Ads API: **$1 minimum spend per ad**
- No per-API-call charges

**Best Practices:**
- Account warmup: 7-14 days required for new accounts (30-60 min/day native usage)
- Posting frequency: 3-5x daily optimal
- Video length: 15-60 seconds (Shorts format)
- Engagement: Video format outperforms statics; trending sounds matter
- Audio is critical: 40% of algorithm weight is sound selection

**Gaps in Official API:**
- ❌ Cannot upload audio/music directly (requires platform-native selection)
- ❌ Comment moderation via API (limited)
- ❌ Limited brand safety/content filtering options

**Recommendation:** Use Postiz for cross-platform handling; direct TikTok API for analytics + hashtag research.

### 2. Instagram

**Official APIs Available:**
- Instagram Graph API (Meta's official, business/creator accounts only)
- Content Publishing API (Reels, feed, Stories, Carousel)
- Instagram Insights API (analytics)
- Instagram Platform API (July 2024+, improved video support)

**Content Publishing Capability:**
- ✅ **Reels upload** (video, up to 90 seconds)
- ✅ **Feed posts** (image, carousel, video)
- ✅ **Stories** (image, video, short-form)
- ✅ **Scheduled posting**
- ✅ **Container model:** Create container → monitor status → publish

**Analytics:**
- Excellent granularity: engagement rate, reach, impressions, saves, shares, traffic
- Breakdown by content type, time posted, hashtags used
- Best times to post (generated by algorithm)

**Hashtag Research:**
- Instagram Graph API: Hashtag Search + IG Hashtag Insights
- Real-time trending hashtags available
- Competitor tag tracking possible

**Rate Limits & Costs:**
- API access: **Free** (requires Meta Developer account + business/creator account)
- Per-call limits: 200 calls per 1 hour (business), 3,600 calls per 24 hours (creator)

**Best Practices:**
- Reels perform 40-67% better than feed posts on Instagram
- Optimal posting: 1-2x daily (Reels), 3-5x for Stories
- Video length: 15-90 seconds (Reels), 1-60 seconds (Stories)
- Hashtag strategy: 3-5 niche hashtags + 5-10 broad hashtags

**Gaps in Official API:**
- ❌ Cannot directly upload audio (must be part of video)
- ❌ Limited Story analytics (basic only)
- ❌ No carousel video support yet (only carousel images)

**Recommendation:** Use Instagram Graph API directly; integrate into Postiz for unified posting.

### 5. Twitter / X

**Official APIs Available:**
- X API v2 (POST endpoint for posting)
- Premium API (enhanced analytics, historical data)
- X Ads API (campaign management)

**Content Publishing Capability:**
- ✅ **Text posts** (up to 280 characters, 300 characters for premium members)
- ✅ **Threads** (connected posts, up to 25 posts per thread)

- Hashtags: Use sparingly (1-2 per post); engagement matters more than reach

- Threads: 5-10 connected tweets drive higher engagement

**Gaps in Official API:**
- ❌ Cannot target specific audiences via API
- ❌ Limited trend detection via free API

### 6. Reddit

**Official APIs Available:**
- Reddit API (via PRAW - Python Reddit API Wrapper, or direct OAuth)
- Subreddit Management API
- Moderation API

**Content Publishing Capability:**
- ✅ **Text posts** (self.subreddit)
- ✅ **Link posts** (URL + description)
- ✅ **Image posts** (single or gallery)
- ✅ **Video posts**
- ✅ **Scheduled posting** (via scheduled sorting or third-party)
- ⚠️ **Cross-posting** (limited, depends on subreddit rules)

**Analytics:**
- Limited official analytics
- Vote count, comment count available immediately
- Engagement metrics (upvotes, downvotes) visible but time-delayed

**Hashtag Research:**
- Reddit uses subreddit tagging, not hashtags
- Keyword research: Search subreddits by topic, analyze post performance
- No official trending API; manual research or third-party tools

**Rate Limits & Costs:**
- API access: **Free** (requires Reddit app registration)
- Rate limits: 60 requests per minute per user
- Per-call charges: **None**

**Best Practices:**
- Community approval: Must understand subreddit culture (will be auto-removed if off-brand)
- Posting frequency: 1-2x per day max (avoid spam flags)
- Content: Authentic, helpful posts drive engagement; overt promotion gets downvoted
- Timing: 10am-2pm UTC performs best (US hours)

**Gaps in Official API:**
- ❌ Cannot schedule posts via Reddit API (must use third-party)
- ❌ Limited community discovery via API
- ❌ No native A/B testing framework
- ⚠️ Strict anti-bot enforcement (Reddit will ban accounts that look automated)

**Recommendation:** Use Reddit API for community-specific strategies; risky for automated mass posting. NOT recommended for Atlas first phase.

**1. OpenAI DALL-E 3 (RECOMMENDED PRIMARY)**
- **Pricing:** $0.040 (1024x1024) / $0.080 (1024x1792) / $0.120 (HD 1024x1792)
- **Pros:** Best realism, safest for commercial use, easiest API integration
- **Cons:** Most expensive, slower generation
- **Use case:** Marketing images that must look professional/photorealistic
- **Typical monthly cost (3x/day posts):** $3.60/day × 30 = **$108/mo**

**2. Stability AI (RECOMMENDED SECONDARY)**
- **Pricing:** $0.01-0.03/image (depending on resolution + model)
  - SDXL: $0.01/img
  - SD3: $0.03/img
- **Pros:** Cheapest, fastest, flexible, good for stylized content
- **Cons:** Less photorealistic, more artistic
- **Use case:** Quick iteration, A/B testing, stylized graphics
- **Typical monthly cost (3x/day posts):** $0.03 × 90 posts = **$2.70/mo**

**3. Midjourney API (OPTIONAL, NOT YET GENERALLY AVAILABLE)**
- **Pricing:** $0.30-2.00/image (not API yet; through Discord)
- **Pros:** Highest quality, best for artistic/brand imagery
- **Cons:** Expensive, limited API access, Discord-only currently
- **Use case:** Hero images, brand showcases
- **Status:** API access not yet released publicly; using web scraping is against ToS

**4. Replicate (OPTIONAL FOR FLEXIBILITY)**
- **Pricing:** $0.015-0.025 per image (depends on model)
- **Pros:** Supports many open-source models (Flux, SDXL, others), good API
- **Cons:** Quality variable by model, less support
- **Use case:** Experimenting with different models, cost-optimized production

**5. Local Deployment (Flux/SDXL on self-hosted GPU)**
- **Pricing:** $0 (API cost), but hardware cost
- **Pros:** Unlimited, no rate limits, privacy-focused
- **Cons:** Requires GPU (expensive upfront), slower
- **Use case:** Large-scale production with high volume
- **Cost:** ~$2,000-5,000 GPU upfront + electricity

### Recommendation for Atlas

**Primary:** OpenAI DALL-E 3 (quality + safety)  
**Secondary:** Stability AI (cost optimization + experimentation)  
**Fallback:** Replicate (flexibility)  
**Combined monthly cost (90 images/mo, 70% DALL-E 3, 30% Stability):** ~$77/mo

## PART 4: AUDIO & MUSIC APIs

### Voice Generation (TTS)

**ElevenLabs (RECOMMENDED)**
- **Pricing:** 1 character = 1 credit
  - Starter: 10,000 credits/mo = $5 (or free with limit)
  - Pro: 100,000 credits/mo = $99
  - *For 3x daily voiceovers (~5 min script), needs ~10,000 credits/mo = $5-10*
- **Quality:** Excellent, natural-sounding
- **API:** Mature and stable
- **Pros:** Best TTS quality, large voice library, easy API
- **Cons:** Character-based pricing (longer scripts = more cost)

**Google Text-to-Speech (ALTERNATIVE)**
- **Pricing:** $4 per 1M characters (very cheap)
- **Quality:** Good but more robotic
- **Use case:** If budget is primary concern

### Music Licensing

**Epidemic Sound (RECOMMENDED FOR SOCIAL)**
- **Pricing:** $9.99/mo (personal) or $99/mo (commercial use)
- **Pros:** Unlimited downloads, pre-cleared for YouTube/TikTok, large catalog
- **Cons:** Limited broadcast rights (TV/cinema requires additional fees)
- **Best for:** Social media, YouTube, streaming platforms
- **Monthly cost:** $99 (commercial)

**Artlist (GOOD FOR COMMERCIAL)**
- **Pricing:** $9.99/mo (personal) or $199/mo (Pro commercial)
- **Pros:** Cheaper than Epidemic for commercial, includes stock footage/templates
- **Cons:** Broadcast use requires PRO negotiation
- **Best for:** Content creators planning broader distribution
- **Monthly cost:** $199 (commercial)

**Native Platform Audio (BEST PRICE)**
- **TikTok:** Built-in sound library (free)
- **YouTube:** Audio Library (free, curated music)
- **Instagram:** Built-in audio (free)
- **Benefit:** No licensing issues, algorithm-friendly
- **Downside:** Limited variety, heavily used

### Recommendation for Atlas

**Strategy:** Use native platform audio first (free), supplement with Epidemic Sound for variety.
- **Monthly cost:** $99/mo (Epidemic commercial)
- **Voiceovers:** ElevenLabs $10/mo (light usage) or skip for MVP

## PART 6: POSTING INFRASTRUCTURE

### Postiz Deep Dive (CURRENT CHOICE)

**What is Postiz?**
- Open-source social media scheduler
- Supports 28+ platforms (TikTok, Instagram, YouTube, LinkedIn, Twitter, Reddit, Facebook, etc.)
- Self-hosted or cloud (postiz.pro)
- API available for automation

**Features:**
- ✅ Multi-platform scheduling
- ✅ Content calendar
- ✅ Analytics aggregation (per platform)
- ✅ Team collaboration
- ✅ Webhook integration
- ✅ Custom integrations

**Pricing:**
- **Cloud (postiz.pro):** $19-99/mo (depends on platform count + scheduling)
- **Self-hosted:** Free (open-source) + hosting costs ($10-50/mo)

**Pros:**
- Cheapest option for multi-platform posting
- Open-source (can customize)
- API-first design
- Supports all major platforms

**Cons:**
- Analytics are aggregated, not deeply analyzed
- Requires platform integrations (OAuth setup for each)
- No AI content generation built-in
- Community-driven support (not enterprise)

### Alternatives Considered

**Buffer**
- **Pricing:** $5-40/mo per platform
- **Pros:** Simple, reliable, good analytics
- **Cons:** Expensive at scale; less feature-rich than Postiz

**Later**
- **Pricing:** $25-100/mo
- **Pros:** Instagram-optimized, beautiful interface
- **Cons:** Not ideal for developers, less flexible API

**HubSpot**
- **Pricing:** $50-3,200/mo
- **Pros:** Full CRM + marketing automation
- **Cons:** Overkill for just social posting, expensive

### Recommendation

**Use Postiz (postiz.pro cloud or self-hosted)**
- **Cost:** $50-99/mo (cloud, all platforms)
- **Integration:** Already used for Larry; proven reliable
- **API:** Mature enough for Atlas integration

## PART 7: ANALYTICS AGGREGATION

### Challenge

Each platform has different metrics:
- **TikTok:** Views, completion rate, engagement
- **Instagram:** Reach, impressions, saves, shares
- **YouTube:** Watch time, AVD, CTR
- **Twitter:** Impressions, engagement, traffic

**1. Platform-Native APIs (RECOMMENDED FOR MVP)**

- **Use case:** Brand monitoring, not for Atlas

### Recommendation for Atlas

**MVP:** Platform-native APIs
- TikTok, Instagram, YouTube direct
- Postiz API for secondary platforms

**Phase 2:** Integrate Outstand if scaling beyond 10 platforms

**Cost:** Free initially

## PART 10: KEY DEPENDENCIES & GOTCHAS

### Critical Constraints

1. **TikTok Account Warmup (7-14 days mandatory)**
   - New accounts need natural usage before automated posting
   - 30-60 min/day scrolling, liking, following

2. **Audio Selection (TikTok)**
   - API cannot directly select trending audio
   - Posts must go to SELF_ONLY drafts
   - User manually adds audio from TikTok's library before publishing

3. **OAuth Token Expiry**
   - Instagram, YouTube, LinkedIn tokens expire
   - Refresh token handling required
   - Automatic refresh + user notification for manual re-auth

4. **Rate Limits (Per Platform)**
   - TikTok: 200 calls/hour
   - Instagram: 200 calls/hour
   - YouTube: 10,000 units/day (not per-call)
   - X: Depends on tier ($42 tier very limited)

5. **Content Approval (Some Platforms)**
   - LinkedIn: May hold posts for review (especially links)
   - Facebook: Community Standards enforcement
   - Reddit: Subreddit-specific approval queues

6. **Analytics Delay**
   - TikTok: 2+ hours before analytics available
   - Instagram: 24 hours for full metrics
   - YouTube: 24-48 hours for detailed analytics

## PART 11: SECURITY & CREDENTIAL MANAGEMENT

### Best Practices Implemented

1. **Never store API keys in code**
   - Use environment variables (.env file)
   - Use secrets managers (1Password, Vault, AWS Secrets Manager)

2. **OAuth Token Rotation**
   - Automatic refresh before expiry
   - Secure storage of refresh tokens

3. **Rate Limiting Protection**
   - Queue requests to avoid hitting limits
   - Exponential backoff on 429 errors

4. **Audit Logging**
   - Log all API calls (for debugging + cost tracking)
   - Never log sensitive credentials

5. **Platform Compliance**
   - Follow each platform's automation guidelines
   - Avoid patterns that trigger bot detection
